Permission (ACL)

เลือก Role ด้านซ้าย — ด้านขวาเป็น 3 ชั้นคงที่ เรียงลงเสมอ: TOPIC → GROUP → CLUSTER (ไม่มี dropdown เลือกประเภท resource). แต่ละเทมเพลตจะใช้บางชั้นเท่านั้น — ชั้นที่ไม่ใช้จะแสดงข้อความว่าง. โฟกัสหนึ่ง topic หลักต่อรอบ — หลาย topic ใช้ Add ACL ภายหลัง. Topic จากขั้นก่อน sync เข้าชื่อ topic หลัก. LITERAL / PREFIXED ปรับได้ตามแถวที่แก้ชื่อได้. การรัน ACL จริงใช้ bootstrap จาก server ตาม environment.

คลิกการ์ดหนึ่งใบ — ฝั่งขวาจะจัดชั้น TOPIC → GROUP → CLUSTER ให้ตรงเทมเพลต ไม่ต้องเลือกประเภท resource เอง

ชื่อ topic และ consumer group (รูปแบบ ชื่อผู้ใช้ + -cg) มาจากขั้นก่อนหน้า — ปรับ LITERAL / PREFIXED และคลิก operation ได้ที่นี่


          

          

            Kafka ACL reference (English)
            

              
ResourceOperationDescription

              
                
TopicReadConsume messages from the topic. Required for consumers.

                
WriteProduce messages to the topic. Required for producers.

                
DescribeView topic metadata (partitions, offsets). Usually needed with Read/Write.

                
DescribeConfigsView topic configuration.

                
Create, Alter, AlterConfigs, DeleteAdmin-level: create/alter/delete topics. Do not grant to normal clients.

                
AllAll of the above.

                
Consumer GroupReadJoin group and consume. Included in Consumer template.

                
DescribeView group metadata (members, offsets). Optional.

                
DeleteDelete the consumer group. Optional; grant only if needed.
