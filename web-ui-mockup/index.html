Identification & Topic

Client system name is used for the pack folder and filename. Topic must exist in the cluster.

Used for pack folder and filename.
Must exist. Load list below or type the name.

Kafka username

Must not already exist. See existing users below — do not reuse.

Letters, numbers, dots, underscores, hyphens only.

Permission (ACL)

Topic ACL and optional Consumer Group permissions. Consumer Group Read is added by default (required for consume/produce).

Read = consume messages. Write = produce messages. Describe = view metadata. Client = Read + Write + Describe + DescribeConfigs.

Consumer Group ACL

Consumer Group Read is added automatically (required for consume and typical produce flows). Optionally add:

Kafka ACL reference (English)
ResourceOperationDescription
TopicReadConsume messages from the topic. Required for consumers.
WriteProduce messages to the topic. Required for producers.
DescribeView topic metadata (partitions, offsets). Usually needed with Read/Write.
DescribeConfigsView topic configuration.
Create, Alter, AlterConfigs, DeleteAdmin-level: create/alter/delete topics. Do not grant to normal clients.
AllAll of the above.
Consumer GroupReadJoin group and consume. Added by default (required for consume).
DescribeView group metadata (members, offsets). Optional.
DeleteDelete the consumer group. Optional; grant only if needed.

Passphrase for .enc file

Used to encrypt the credential pack. You need this to decrypt on your machine.

Execute

Review and create. Backend will call the shell script (path in config) with your inputs.
