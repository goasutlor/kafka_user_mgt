Kafka & runtime

Compose: typically /opt/kafka-usermgmt via runtime volume.

For Confluent on OpenShift, oc get secret -n <namespace> lists many Kafka-related secrets. This application updates the secret that holds the plain SASL user list (key plain-users.json). For standard Confluent deployments that name is fixed: kafka-server-side-credentials. Others (e.g. kafka-admin-client-credentials, kafka-client-side-credentials, listener apikeys) are for different purposes — you do not type those here unless your platform uses a different secret name for the same role.

Kafka client .properties (สำคัญ)
ไฟล์ต้องอยู่บน host ใต้โฟลเดอร์ที่ mount เป็น {runtimeRoot}/configs/ (เช่น Podman: ~/kafka-usermgmt-runtime/configs/).
ช่องด้านบนคือ แค่ชื่อไฟล์ ไม่ใช่ path เต็ม — ตัวอย่าง: ถ้าคุณมี client-ssl-external.properties ให้ใส่ชื่อนั้น แล้วคัดลอกไฟล์ไปที่ …/configs/client-ssl-external.properties
พาธในไฟล์ (เช่น ssl.truststore.location) ต้องชี้ไปที่ไฟล์ที่ container เห็นได้ — แนะนำเก็บ .jks ไว้ใต้ configs/ แล้วใช้ path แบบ /opt/kafka-usermgmt/configs/client.truststore.jks (ให้ตรงกับ runtime root ที่ mount)
