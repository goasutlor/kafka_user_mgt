Kafka (ต่อ environment)

แต่ละแถวด้านล่างสัมพันธ์กับ environment ที่ตั้งในขั้น 1 (เดียวกับชื่อใน header ของ portal). กรอก bootstrap เป็น host:port ของ Kafka TLS route สำหรับ project นั้น. Truststore + SASL ด้านล่างใช้ ร่วมกันทุก environment (credentials เดียวกันหลาย namespace). แถวแรกที่กรอก bootstrap จะเติมช่อง Default bootstrap ด้านล่างให้อัตโนมัติ (แก้ได้). เมื่อบันทึก Setup: ระบบจะสร้างไฟล์ kafka-client-<id>.properties และ kafka-client-master-<id>.properties ใต้ runtime configs/ ให้ตรงกับแต่ละ environment (bootstrap ต่อแถว) — ไม่ต้องคัดลอกมือถ้าใช้ wizard ครบ

Bootstrap ต่อ environment — host:port (TLS)
Per-environment bootstrap (legacy UI)

Templates vs full config: (1) Leave the SSL/SASL section below empty — we create templates (with CHANGE_ME) if missing: default pair plus per-env kafka-client-<id>.properties when multi-environment is on. (2) Fill it — we write full kafka-client.properties, kafka-client-master.properties, and the same for each environment id (shared truststore/SASL, different bootstrap.servers per row). Recommended: copy your .jks onto the runtime volume (e.g. configs/client.truststore.jks), choose File on server, enter truststore password + SASL — Setup checks the file is present and runs keytool -list on Verify/Save. Use Paste only if you cannot place a file on the mount yet.
